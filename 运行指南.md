## 🚀 如何运行主脚本

由于 WSL 环境缺少依赖，建议在 **Windows PowerShell** 中运行：

### 步骤 1: 打开 PowerShell
- 按 `Win + X`，选择 "Windows PowerShell" 或 "终端"
- 或者右键点击开始按钮，选择 PowerShell

### 步骤 2: 切换到桌面目录
```powershell
cd C:\Users\Administrator\Desktop
```

### 步骤 3: 安装 Python 依赖（首次运行）
```powershell
pip install selenium openpyxl
```

如果没有 pip，先安装 Python：
- 下载：https://www.python.org/downloads/
- 安装时勾选 "Add Python to PATH"

### 步骤 4: 运行脚本
```powershell
python deepseek_listing_generator_v4.py
```

### 步骤 5: 选择处理范围
- 输入 `2` 测试前 3 个（推荐）
- 输入 `1` 处理全部
- 输入 `3` 自定义范围

### 步骤 6: 确认 DeepSeek
浏览器会自动打开 DeepSeek 网页
- 确保已登录
- 确认 Prompt 已配置好
- 按回车继续

## ⚙️ 配置说明

脚本开头的配置区可以修改：

```python
EXCEL_PATH = r"C:\Users\Administrator\Desktop\test2.xlsx"  # 输入文件
OUTPUT_PATH = r"C:\Users\Administrator\Desktop\test2_output.xlsx"  # 输出文件
DATA_START_ROW = 7  # 数据起始行
WAIT_TIMEOUT = 120  # 等待 DeepSeek 回复的最大秒数
MAX_RETRIES = 3  # 最大重试次数
DEBUG_MODE = True  # 是否保存调试文件
```

## 🐛 常见问题

### 1. Selenium 版本问题
如果报错 "SessionNotCreatedException"，更新 EdgeDriver：
```powershell
pip install --upgrade selenium
```

### 2. 浏览器找不到
确保已安装 Microsoft Edge 浏览器

### 3. 解析失败
检查 `debug_v4_rowX.txt` 文件，查看原始回复内容

### 4. Excel 打不开
确保 Excel 或 WPS 已安装并关闭 test2.xlsx
